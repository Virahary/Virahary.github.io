### 1.3 链接
（1）插入本地图片

格式：

\!\[图片描述](图片路径 "title\")

\!\[图片描述](图片路径 )

这里的“图片描述”不是必要的

例子：

\!\[我们拿Elon举例子](/_posts/pics/pics1/Elon.png "This is Elon!")

![我们拿Elon举例子](pics/pics1/Elon.png "This is Elon!")

注意：对于编辑器上的插件来讲，/_site/pics/pics1/Elon.png的路径，才能正确显示出图片；而对于jekyll构建的效果来讲，得拿/pics/pics1/Elon.png作为路径。

（2）插入互联网上的图片

格式：

\!\[图片描述](图片网络路径)

注意：这里的图片描述可以不写

(3)自动链接

Markdown支持以比较简短的自动链接形式来处理网址和电子邮箱，只要是用<>包起来，Markdown就会自动把它转成链接，也可以是直接显示成链接形式。<www.baidu.com>

### 1.4 分割线
你可以在一行中用三个以上的星号（*）、减号（-）、底线（_）来建立一个分隔线，行内不能有其他东西。你也可以在这些符号中间插入空格。
三种输入方式创造出来的分割线一样的，或许作者只是想让我们在写代码的时候区分开来。

### 1.5 代码块
（1）利用缩进（与前后文空一行）

    }

（2）行内式
如果需要在一行内引用代码，只需要用反引号``引起来就好（在Esc下面，和~同一个键位）

（3）多行代码块与语法高亮
在需要高亮的代码块的前一行及后一行使用三个单反引号“ ` ”包裹，就可以了

    ```javascript {highlight=10}
    class demo(){
        public static void test(){
        }
    }
    ```

（4）代码块里面包含html代码
在代码块里面，&、< 和 > 会自动转成HTML实体，这样的方式让你非常容易使用Markdown插入范例用的HTML原始嘛，只需要复制贴上，剩下的Markdown都会帮你处理。

示例如下：
```

            <th rowspan="2">值班人员</th>
            <th>星期一</th>

        </tr>
        <tr>
            <td>李强</td>
            <td>张明</td>
            <td>王平</td>
        </tr>
    </table>
```

在引用的文本之前加上>符号，以及一个空格就行了；
如果只输入了一个>符号就会产生一个空白的引用。
（1）基本使用

>真的嘛？
>

（2）引用的嵌套

>>>引用一
>>引用二
>引用三
>>>
>>>引用四

（3）引用其他要素
引用的区块内也可以使用其他的Markdown语法，包括标题、列表、代码区块等等内容。

### 1.7 列表
（1）无序列表
使用*，+,- 表示无序列表
**注意符号后面一定要有一个空格起到缩进的作用**

* 列表文字
* 列表文字
* 列表文字

（2）有序列表
使用数字和一个英文句点表示有序列表
**注意英文句点后面一定要有一个空格，起到缩进作用。**
1. 张鑫阳
2. 陈卓

（3）有序列表、无序列表可以混用

（4）列表可以与其他元素混合使用

>（1）加粗效果不能直接用于列表标题中，但是可以嵌套在列表里面混合使用
>（2）列表中包含代码块（前面加2个tab或者8个空格，并且需要空一行，否则不显示）

* **（1）的说明：\* \*\*（1）的说明\*\***

（5）注意事项
再使用列表的时候，只要是数字后面加上英文的点，就会无意间产生列表

### 2.4 字体、字号和颜色

Markdown是一种可以使用普通文本编辑器编写的标记语言，通过类似HTML的标记语法，它可以使普通文本内容具有一定的格式。但是它本身是不支持修改字体、字号与颜色等功能的！

\<font face="黑体">我是黑体字\</font>
<font face="黑体">我是黑体字</font>

\<font color=#0099ff size=7 face="黑体">我是黑体字\</font>
<font color=#0099ff size=7 face="黑体">我是黑体字</font>

\<font color=gray size=7 face="黑体">也可以使用已知颜色的名字\</font>
<font color=gray size=5 face="黑体">也可以使用已知颜色的名字</font>

#### 2.5.5 注脚

一般需要在添加注脚的文字后加上脚注名字[^注脚名字]，称为加注。
然后在文本的任意位置（一般在最后）添加脚注，脚注前必须有对应的脚注名字。

示例如下：
使用Markdown[^1]可以效率的书写文档，直接转换成HTML[^2]。

    脚注被自动搬运到最后面，请到文章的末尾查看，并且脚注后方的链接可以直接跳转回加注的地方。

[^1]:Markdown是一种纯文本标记语言
[^2]:HyperText Markup Language 超文本标记语言

### 2.6 背景色

 Markdown本身不支持背景色设置，需要采用内置html的方式实现：借助 table, tr, td 等表格标签的 bgcolor 属性来实现背景色的功能。举例如下：

<table><tr><td bgcolor=orange>背景色是：orange</td></tr></table>
